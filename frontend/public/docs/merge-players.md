# Merge vs. rename players

Two different fixes for two different problems — choose the right one.

## Rename / alias (same person, wrong text)

Use this when the player already exists once and you just want to change how their
name is shown, or record an alternate spelling.

- Admin → **Players** → open the player → edit the **display name** (or add an
  **alias**).
- Their history stays intact; only the label changes.

## Merge (the same person exists twice)

Use this when one human has **two player records** — e.g. signed up under a
nickname one week and their full name another — and their results are split.

- Admin → **Players** → use the **merge** tool to fold the duplicate into the
  canonical player.
- All tournament results from the duplicate move onto the kept player, and the
  duplicate is removed.

> Rule of thumb: **rename** changes a label, **merge** combines two records.
> Merging cannot be undone, so double-check you've picked the right "keep" player.
